What makes it different

A DFIR analyst that can't lie — and can't touch the evidence.

Capabilities the baseline Protocol SIFT — and most competitors — don't have, working together end to end.

Read-only by construction

The MCP server exposes only typed read functions — no shell, write, or delete tool exists for the agent to call. Spoliation is architecturally impossible, not prompt-requested.

30
read-only tools, zero write paths

Mechanical hallucination gate

Every finding is re-checked against the captured raw output of the tool it cites. Ungrounded claims are quarantined — never reported as fact. This is the GTG-1002 failure mode, engineered out.

0 fabricated findings

Adversarial red-team

A panel of skeptics challenges every finding — upheld, demoted, or refuted — removing false positives like benign DNS.

Cross-source correlation

Disk vs memory: hidden processes (psscan ≠ pslist), orphan connections, bad parents that single-source misses.

Hash-chained audit + deterministic replay

A tamper-evident chain of custody where each record's hash covers the prior. The full finding set re-derives from the log alone — court-admissible reproducibility (FRE 901 / Daubert).

How it works

The model proposes. The graph disposes.

A read-only MCP server under a LangGraph pipeline. The control flow, the verdicts, and the loop bound are all code — not prompts.

Architectural, not prompt-based

Guardrails you can prove, not ones you hope for.

"Please don't modify the evidence" is a wish. GLASSBOX removes the capability — and ships a one-click self-test that proves every boundary holds.

Prompt-based (the baseline)
  • ✕ "Read-only mode" is a system-prompt instruction the model can be talked around.
  • ✕ A general shell tool with an allow-list — many "allowed" tools can still write.
  • ✕ Findings are whatever the model asserts; no mechanical check.
  • ✕ Retry loops with no hard bound can spiral.
GLASSBOX (architectural)
  • ✓ No write / shell / delete tool exists in the MCP surface — spoliation is impossible.
  • ✓ Path traversal blocked in code; before/after SHA-256 proves zero modification.
  • ✓ Every finding re-verified against captured output, then red-teamed.
  • ✓ Loop bounded by an in-code max-iterations counter + recursion limit.

Hash-chained audit

Every tool call, message and verdict linked by SHA-256. Any insertion, deletion or edit breaks the chain and is detected on re-verify.

Deterministic replay

The complete finding set re-derives from the audit log + captured output alone — a different examiner reaches identical conclusions.

Guardrail self-test

One click actively tests six architectural boundaries for bypass — no-write-tool, path traversal, evidence read-only, audit tamper, hallucination, HMAC.
